# Product Requirements Document

## 1. 사용자

- 중학생 또는 고등학교 기초 물리 학습자
- 과학 교사
- 방과후/에듀테크 실험 수업 운영자

## 2. 사용자 스토리

- 학생으로서, 회로를 직접 연결하면서 왜 전구가 켜지는지 알고 싶다.
- 학생으로서, 틀렸을 때 정답을 바로 듣기보다 내가 어디서 헷갈렸는지 알고 싶다.
- 교사로서, 학생들이 어떤 오개념을 반복하는지 수업 후 확인하고 싶다.
- 개발자로서, 학습 로그를 분석해 피드백 정책을 개선하고 싶다.

## 3. MVP 기능 우선순위

| 우선순위 | 기능 | 완료 조건 |
|---|---|---|
| P0 | XR 부품 배치/연결 | 학생이 배터리, 전구, 전선으로 닫힌 회로를 만들 수 있음 |
| P0 | 회로 그래프 변환 | Unity 연결 상태가 JSON 그래프로 서버에 전송됨 |
| P0 | 회로 판정 | 닫힌 회로와 직렬/병렬 여부를 서버가 반환함 |
| P0 | 피드백 정책 | MRR 기반 피드백 모드가 선택됨 |
| P1 | LLM 설명 분석 | 학생 설명에서 개념 태그와 오류 후보를 추출함 |
| P1 | 반례 시뮬레이션 | 직렬/병렬 비교가 XR에서 시각화됨 |
| P2 | 교사용 대시보드 | 세션별 오개념과 수정 성공률이 표시됨 |

## 4. 비기능 요구사항

- 분석 응답 시간: MVP 기준 2초 이내 목표
- 오프라인 폴백: LLM 실패 시 규칙 기반 힌트 제공
- 개인정보: 익명 세션 ID 기반 저장
- 교실 사용성: 5분 이내 튜토리얼 완료
- 시연 안정성: 네트워크 장애 시 데모 데이터 모드 지원
